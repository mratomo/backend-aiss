Visión General del Sistema

  • Sistema distribuido para gestión de conocimiento con capacidades RAG (Retrieval Augmented Generation)
  • Arquitectura de microservicios altamente modular y escalable
  • Implementa el Model Context Protocol (MCP) para estructuración de información
  • LangGraph Futura implementación para flujos de trabajo de IA avanzados
  • Structured Output para generar respuestas en formatos específicos

Tecnologías principales

  • Go: Servicios core de alto rendimiento
  • Python: Servicios MCP y procesamiento de lenguaje
  • MongoDB: Almacenamiento de datos
  • Qdrant: Base de datos vectorial
  • MinIO: Almacenamiento de objetos
  • Comunicación: APIs REST

Componentes: Servicios Core

User Service

  • Gestión completa de usuarios y autenticación
  • JWT con tokens de acceso y refresco
  • Control de acceso basado en roles (RBAC)
  • Gestión de perfiles y preferencias
  • Auditoría de accesos y seguridad

Document Service

  • Gestión del ciclo de vida de documentos
  • Procesamiento e indexación de documentos
  • Extracción de texto y metadatos
  • Chunking inteligente de contenido
  • Almacenamiento seguro en MinIO

API Gateway

  • Punto único de entrada al sistema
  • Autenticación y enrutamiento de solicitudes
  • Balanceo de carga y limitación de tasas
  • Monitorización y logs de acceso
  • Configuración CORS y seguridad de cabeceras

Los servicios core están implementados en Go para maximizar rendimiento y escalabilidad en operaciones críticas.

La arquitectura orientada a microservicios permite:

  • Despliegue independiente de componentes
  • Escalado horizontal específico por servicio
  • Resiliencia ante fallos parciales

Plan de Implementación: LangGraph

Implementación Futura

La arquitectura está preparada para incorporar LangGraph para mejorar los flujos de razonamiento y resolver problemas complejos.

Características planeadas

  • Orquestación de flujos de trabajo de IA complejos
  • Agentes con memoria y estado persistente
  • Encadenamiento dinámico de procesos LLM
  • Trazabilidad completa del razonamiento
  • Pensamiento multicapa para problemas estructurados

Estado Actual: Flujo Secuencial

El sistema actual implementa un flujo RAG secuencial que funciona correctamente para la mayoría de los casos de uso:

  1. Recepción de consulta
  2. Procesamiento y generación de embedding
  3. Búsqueda de contexto relevante
  4. Formulación de prompt enriquecido
  5. Generación de respuesta por LLM

Próximos pasos

Para implementar LangGraph se requiere:

  • Integrar la biblioteca en requirements.txt
  • Definir nodos de procesamiento específicos
  • Diseñar flujos para los casos de uso prioritarios
  • Implementar persistencia de estado entre llamadas
  • Desarrollar sistema de monitoreo del flujo

Seguridad: Configuración de Red

Internet/Red Externa
Frontend (Nginx/Proxy)
Los usuarios se conectan aquí (puerto 80/443)
API Gateway
No expuesto externamente, solo accesible desde la red Docker
Servicios Internos
Solo accesibles dentro de la red Docker

Ventajas del modelo de red

  • Aislamiento completo de servicios internos
  • Red bridge privada entre contenedores
  • Única entrada externa a través del proxy
  • Superficie de ataque mínima
  • Comunicación TLS para endpoints externos
  • Segmentación de tráfico por servicio

Modelo de Seguridad Multicapa

Perímetro de Red (Firewall + Proxy)
API Gateway (Rate limiting + CORS + Validación)
Autenticación y Autorización (JWT + RBAC)
Seguridad a Nivel de Servicio (Input validation + Logging)
Seguridad a Nivel de Datos (Encryption + Access control)

Implementación de seguridad por capa

  • Red: Aislamiento por contenedor
  • API: Rate limiting, CORS estricto
  • Auth: JWT con tokens de acceso/refresco
  • Servicio: Verificación host keys SSH
  • Datos: Credenciales cifradas con bcrypt
  • Terminal: Validación de comandos

Escalabilidad

  • Servicios stateless para escalado horizontal
  • Bases de datos configurables para replicación
  • API Gateway con balanceo de carga
  • Procesamiento por lotes para operaciones intensivas
  • Particionamiento de datos por cliente/área

Optimizaciones

  • Caché de embeddings: Reutilización para textos similares
  • Optimizaciones GPU: Precisión FP16, batch processing
  • Pool de conexiones: Reutilización eficiente
  • Buffering adaptativo: Ajuste según la carga
  • Compresión de datos: Reduce transferencia

Caso 1: STRATCOM - Comunicación de Prensa

Elaboración Asistida de Comunicados

1. Solicitud de Información

El responsable de STRATCOM solicita información sobre directrices de comunicación para un incidente específico.

2. Recuperación de Contexto

El sistema recupera:

  • Playbooks de comunicación en crisis
  • Plantillas oficiales aprobadas
  • Comunicados previos similares
  • Mensajes clave para stakeholders

3. Análisis de Información

RAG Agent analiza y extrae:

  • Tipo de incidente y mensajes apropiados
  • Audiencias objetivo y canales adecuados
  • Nivel de detalle técnico apropiado
  • Cadenas de aprobación requeridas

4. Generación y Formato

El sistema genera un comunicado oficial con Structured Output, formateado según plantillas institucionales, listo para revisión y aprobación.

15

Caso 3: Resolución de Anomalía en Servicio

Soporte Interactivo en Resolución de Incidentes

1. Detección y Consulta Inicial

Un operador detecta una anomalía en un servicio y consulta al sistema información detallada sobre el servicio afectado.

2. Recuperación de Información Contextual

El sistema consulta su base de conocimiento y proporciona:

  • Máquina donde se aloja el servicio
  • Relaciones con otros servicios
  • Nivel de criticidad
  • Parámetros de configuración
  • Documentación técnica relevante

3. Acceso por Terminal e Inspección

El operador utiliza el Terminal Service para conectarse al servidor y acceder a logs y datos de diagnóstico del servicio afectado.

4. Análisis y Resolución

El sistema analiza los datos recuperados y proporciona:

  • Identificación precisa del problema
  • Causas posibles basadas en patrones conocidos
  • Soluciones recomendadas con comandos específicos
  • Métodos de verificación para confirmar la resolución
